### 第 1 周（1–7 天）——环境 + 基础语法

第1天：环境搭建
安装 Python、配置 VS Code（或 PyCharm），创建虚拟环境，学会使用 pip，运行第一个脚本 hello.py
练习：写并运行几个 print、变量、注释
第2天：基本数据类型与运算
数字、字符串、布尔、输入/输出、格式化字符串
练习：做一个小脚本，把用户输入保存到文件
第3天：控制流
条件语句 if/else、循环 for/while、break/continue
练习：写一个猜数字小游戏
第4天：数据结构
list、tuple、dict、set 的用法与常用操作
练习：实现一个简单的联系人存储在 dict，并能增删查改
第5天：函数与模块
def、参数、返回值、模块导入、main
练习：把前几天的脚本重构为函数并放到模块里
第6天：文件 I/O 与异常处理
open/read/write、with context、try/except/finally、自定义异常
练习：写日志到文件并处理文件不存在的情况
第7天：复习 + 小测验
综合练习：小程序（例如：命令行待办清单，读写本地文件）
第 2 周（8–14 天）——进阶语法、面向对象、标准库

第8天：面向对象编程（OOP）基础
类、对象、方法、属性、构造函数、继承、封装
练习：将联系人管理改成类实现
Day 9：常用标准库与第三方包使用
Datetime、os、pathlib、json、logging;安装与使用 requests
练习：用 requests 调用公开 API（例如天气接口）并保存结果
第10天：包管理与虚拟环境进阶
requirements.txt、pip freeze、venv、简介诗歌
练习：用 requirements.txt 管理依赖
第11天：函数式工具与迭代器、高阶函数
map/filter/reduce、列表推导式、生成器、装饰器基础
练习：写装饰器记录函数运行时间
第12天：单元测试基础（pytest）
写简单测试函数，断言，运行 pytest
练习：为你之前写的模块写 3 个测试
第13天：Git 与 GitHub 基本流程
git init / add / commit / branch / push，写好 README
练习：把当前代码推到 GitHub，写第一版 README
第14天：小项目决定 & 设计（选一个并写出需求/数据库设计）
推荐项目（任选其一）：
A. 待办事项 Web 应用（Flask + SQLite）
B. 个人账单/记账 CLI 或 Web（CLI 用 Typer，Web 用 Flask）
C. 数据抓取 + 分析（爬虫抓取并用 pandas 分析）
练习：写出功能列表、数据模型（表结构）、界面草图或命令交互流程
第 3 周（15–21 天）——开始实现项目核心（以 Flask Web 应用为示例）

第15天：搭建项目骨架
创建包结构：app/， templates/， static/， models.py， routes.py;配置 venv、requirements.txt
练习：能启动一个基本 Flask “Hello” 页面
Day 16：数据库与模型（SQLite + SQLAlchemy）
设计模型、创建迁移（手动或用 Flask-Migrate 可选）
练习：实现增删改查（CRUD）接口（先通过 shell 测试）
第17天：实现 Web 表单与模板（Jinja2）
创建表单页面、新增/编辑/删除功能
练习：把 Bootstrap 加入，做基础样式
第18天：用户输入验证与错误处理
WTForms（可选）或自己在后端验证，友好错误提示
练习：表单校验并返回提示
第19天：用户认证（可选，基础版）
简单登录/登出（会话 cookie），或用第三方登录（可选）
练习：限制某些页面需要登录
Day 20：API 端点与前后端交互（AJAX）
提供 JSON API，练习用 fetch 或 axios 调用
练习：做一个异步删除或分页
第21天：写测试（端点 & 数据库）
用 pytest 测试重要路由与模型逻辑，配置测试数据库
练习：至少 3 个测试通过
第 4 周（22–30 天）——完善、打包、部署与展示

第22天：日志、配置与环境变量
使用 python-dotenv 管理配置、设置 logging
第23天：错误监控与异常处理（用户友好页面）
404/500 页面，自定义异常日志
第24天：优化与性能小技巧
减少 DB 查询、分页、缓存静态资源（简单说明）
第25天：写文档与 README（包含项目说明、安装与运行步骤）
练习：在 README 中写如何部署、如何贡献
第26天：容器化（可选）与生产运行
写 Dockerfile，示例 docker build/run;或写 gunicorn 启动配置
第27天：部署（把应用放到云或演示服务器）
可选平台：Render / Railway / 自己的 VPS（选择一个），或用 Docker 在远程机器运行
练习：部署并能从公网访问（或通过端口映射）
第28天：收尾功能与用户体验打磨
修复已知 bug、补充表单校验、细化样式
第29天：准备演示与录屏 / 准备项目说明文档
写项目演示流程、准备示例数据
第30天：回顾 + 未来计划
把项目推到 GitHub（带 README、LICENSE），写下一步学习计划（如：更深入的测试、CI/CD、前端框架、异步架构等）
五、每日练习/作业示例（便于落地）

每天至少提交一次 Git commit，写清楚 commit message
每项新技能至少写一个小脚本或一个测试用例
在第 14 天后：每天把实现进度记录在 README/MILESTONE
六、最终项目建议（示例需求，适合 30 天完成）

项目：个人账单管理 Web 应用（Flask + SQLite）
功能：用户登录、添加账单（时间/金额/分类/备注）、账单列表（分页/筛选）、删除/编辑、导出 CSV、每月统计图表
技术栈：Flask、SQLAlchemy、Bootstrap、Chart.js（或 matplotlib 生成图像）、pytest
可交付物：GitHub 仓库、README、演示视频或截图、已部署实例（如可）
七、学习方法建议（加速你达到目标）

每天至少编码 1 小时，最好能连续 30 天坚持
读文档优先于看文章：遇到问题先查官方文档
多写少改、频繁提交、写清变更说明
把问题贴到社区（掘金/知乎/Stack Overflow），并把解决过程写成笔记
结对/找人 review：如果有人能 review 你的 PR 会更快进
